# Security Policy

## Supported versions

Wind Tunnel is pre-1.0. Security fixes are made against the latest released
minor version; older minor versions may not receive backports. Upgrade to the
latest release before reporting a problem that may already be fixed.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability.

Use GitHub's private vulnerability reporting:

<https://github.com/syntropy-systems-oss/windtunnel/security/advisories/new>

Include the affected version, impact, reproduction steps or a proof of concept,
and any suggested remediation. Remove credentials, customer data, and other
sensitive information from traces and examples.

Maintainers will acknowledge reports on a best-effort basis, investigate them
privately, and coordinate disclosure with the reporter when appropriate. This
policy does not create a service-level agreement or bug-bounty program.
