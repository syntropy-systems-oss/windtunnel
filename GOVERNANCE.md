# Governance

Wind Tunnel is an open-source project stewarded by Syntropy Systems, Inc.
Community participation is welcome, and project stewardship remains with the
company.

## Roles

- **Contributors** report problems, propose changes, improve documentation, and
  submit code.
- **Maintainers** review contributions, set technical direction, manage releases,
  and handle security and community moderation. Syntropy Systems appoints and
  removes maintainers.

Current code owners are listed in [`.github/CODEOWNERS`](.github/CODEOWNERS).

## Decisions

Maintainers seek consensus through issues and pull requests when practical.
They make final decisions based on the project's scope, compatibility promises,
maintenance cost, security, and long-term health. Significant public API or
architecture changes should be discussed in an issue before implementation.

Syntropy Systems retains final authority over project direction, releases,
security response, project infrastructure, and use of its trademarks. This does
not transfer ownership of outside contributions; accepted contributions remain
licensed under Apache 2.0.

## Releases and security

Maintainers publish releases through the automated process documented in
[`CONTRIBUTING.md`](CONTRIBUTING.md). Security reports follow
[`SECURITY.md`](SECURITY.md) and may be handled privately until a fix and
coordinated disclosure are ready.

## Changes to governance

Governance changes use the same public pull-request process as other project
changes. Syntropy Systems approves changes to this document.
